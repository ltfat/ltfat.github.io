Detailed Description

Internally, heap memory allocation in libltfat is done using functions in this module.

By default, malloc/free functions from FFTW3 are used. FFTW malloc allocates memory properly aligned.

A custom malloc/free can be registered using ltfat_set_memory_handler.

For convenience, there are versions of functions ltfat_malloc() and ltfat_realloc() which allow allocating double, float, complex double and complex float arrays.

Note
According to the FFTW documentation, the allocated memory is not required to be aligned, but it is strongly recommended to do so.

Increase the size of a memory block

The function will call ltfat_realloc only if nnew > nold and sets the padded elements to 0. Otherwise it does nothing. The function works as ltfat_calloc if ptr==NULL and nold is ignored.

Re-allocate memory block

The function will re-allocate nold element array to nnew element array of the corresponding type. The void* version of the function accepts the sizes in bytes. The input memory block ptr is freed if it is not NULL. The function works as ltfat_malloc(nnew) if ptr==NULL and nold is ignored.

[in]ptrPointer to the input block
[in]noldSize of the input block
[in]nnewSize of the output block
